博客:初心和历程

初心

早就想把博客建立起来了，有了自己的博客，才像是有了自己的家一样。很久都没有写博客了，其实应该写的。如果不写点什么也就代表着自己也没有在学习什么。

有挺多的事情想要坚持下去的，其实说实话很多很多。想要学很多的东西，想要完成自己想做的事情。总是用了一个又一个的本子，可是最后都找不到它们的踪影了。很久之后就遗忘了，就不再捡起那些东西。

想要在这里记录自己的成长，生活，变化。也许会有不同的心情，风景。学到了不一样的知识，看了不一样的书之类的。总之，就是想把生活中的事情一点点都做个记录。

之后还有几件比较想做的事情：

  1. 把我一直想写的那本书写一下，不是写给什么人看吧，大概就是想写这个样子。预计会用到的参考类书目：
  • 编码：隐匿在计算机软硬件背后的语言
  • 计算机组成原理
  • x86汇编语言:从实模式到保护模式
  • 数字逻辑
  • 自动机和图灵机
  • 编译原理
  • 操作系统

  1. 记录一直想做的事情，比如学经济学，去西藏，声乐，舞蹈等等。

  2. 除此之外就是一直保持自己的学习状态了，每天都关注学一些专业上的知识。

  3. 多看书。买了很多的书，但是大部分都还没有看完。希望能多看点书，做个胸有点墨之人，哈哈。

历程

从昨天开始找框架软件来支持博客，到今天终于算是初步可以用了。折腾博客的话，放在之后的工作中完成。最近没有那么多的时间去学相关的知识。比如说css，js等内容都需要往后放一放了。

一开始选择的平台是hexo+github，也是在浏览别人的博客的时候发现作者介绍了自己博客的构建方法。看到了他们的博客之后觉得模版非常好看，然后觉得这个方法可行。所以果断安装了，也跑起来了一个初步的模版。

但是后面发现学习hexo确实存在发布比较慢的问题，又从网上看到他们用hugo。hugo的优点就是小，安装非常方便，但是模版相对hexo少一些。功能上hugo是一个静态网站的模版，所以应该比不上hexo炫酷，但对于写博客来说，认为已经足够了。

hugo+github 搭建博客

hugo和hexo相比，生成站点的速度更快。

hugo是用go语言写成的静态站点生成器，是通用网站框架。

Hugo网站可以在任何地方托管，包括Netlify，Heroku，GoDaddy，DreamHost，GitHub Pages，GitLab Pages，Surge，Aerobatic，Firebase，Google云端存储，Amazon S3，Rackspace，Azure和CloudFront，并且可以与CDN配合使用。Hugo站点运行时无需数据库或依赖于Ruby，Python或PHP。

适合hugo的人:

  1. Hugo适合喜欢在浏览器上使用文本编辑器编写的人。

  2. Hugo适用于那些想要自己编写自己网站代码的人，而不必担心设置复杂的运行时，依赖关系和数据库。

  3. Hugo适用于构建博客，公司站点，投资组合站点，文档，单个登录页面或具有数千页的网站的人员。

使用hugo(mac)

  • 安装:brew install hugo
  • 验证: hugo version
  • 创建站点: hugo new site quickstart 实际创建一个quickstart文件夹
  • 添加主题:
git init
git submodule add https://github.com/budparr/gohugo-theme-ananke.git themes/ananke;\
echo 'theme = "ananke"' >> config.toml

需要注意的是

# config.toml文件配置中
# 博客的发布目录。测试时使用如下路径，发布到网络后，要修改为网路地址
baseurl = "https://localhost:1313/"
# 例如改为博主的地址，最后的斜线不能少：  baseurl = "https://dongdongmian.github.io/"
# 尝试了下面的这个命令，表示并不能修改 config.toml，需要手动修改配置文件
hugo --theme=hyde --baseUrl="http://coderzh.github.io/"
